Documentation

Automated Code Review

Review Rules analyze incoming Pull Requests to assign code reviewers according to your process.

Intuitive Comment Interface

Users interact with Cody using comments right in the GitHub PR interface.

Integrates With Other Tools

The review status of every PR is reported as a commit status under the namespace code-review/cody.

How It Works

Cody integrates smoothly into your workflow.

  1. Contributors open Pull Requests like normal.
  2. Cody analyzes incoming Pull Requests and applies your Review Rules.
  3. Code reviewers simply comment on the PR to give their approval.

As reviewers approve and as the PR is updated, Cody tracks the review status and reports it as a commit status on the Pull Request.

Try it Out

You deploy Cody to your own servers and manage it yourself. Cody works particularly well with Heroku but it should work on any host that supports the minimum requirements.

Read the deployment docs for more information.
